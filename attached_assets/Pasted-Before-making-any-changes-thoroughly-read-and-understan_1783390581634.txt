Before making any changes, thoroughly read and understand my existing admin command system, UI structure, audio system, RemoteEvents, and coding style. Reuse the current architecture and only add the functionality described below.

Create a new admin command:

music [Music ID]

Description:
- Typing `music` opens the Music Control menu.
- Typing `music 1841831379` (or any valid Roblox audio ID) immediately plays that audio globally for every player.
- Do not require angle brackets (`< >`) or any special formatting.
- Everything after `music` should be treated as the audio ID.

Examples:
music
music 1841831379
music 7029031068
music 9046435309

--------------------------------------------------
MUSIC COMMAND
--------------------------------------------------

Requirements:
- Play the specified audio globally for every player.
- Stop any currently playing music before starting a new track.
- The music should loop until another track is played or stopped.
- Do NOT display any notifications such as "Now Playing" or similar.
- Music should begin silently without any popups or chat messages.
- Validate the ID before attempting to play it.
- Support changing tracks instantly.

--------------------------------------------------
MUSIC MENU
--------------------------------------------------

Using `music` without any parameters should open a modern, draggable Music Control menu.

Requirements:
- Match the styling of my existing admin menus.
- Opening the command again should toggle the menu.
- The menu should be wider than it is tall.
- Include a search bar that filters songs by ID or title.
- Include a volume slider (0–100%) that updates the global music volume in real time.
- Display the currently playing song at the top.
- Highlight the currently selected song.
- Clicking a song should immediately play it.
- Include a Stop Music button.
- Preserve the selected volume when changing songs.

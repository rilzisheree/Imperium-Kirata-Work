Before making any changes, thoroughly read and understand my existing admin command system, player handling, RemoteEvents, and coding style. Reuse the current architecture and only add the functionality described below.

Create three new admin commands:

heal <target> [amount]

damage <target> <amount>

kick <target> [reason]

--------------------------------------------------
HEAL
--------------------------------------------------

Description:
- Heals the specified player(s).

Usage:
heal <target>
heal <target> <amount>

Examples:
heal Kirata
heal Kirata 25
heal all 50

Requirements:
- Follow the existing admin command framework and coding style.
- Support all existing target selectors (username, display name, me, all, team selectors, etc.) if the command system already provides them.
- If no amount is provided, fully heal the player to their maximum health.
- If an amount is provided, restore only that amount of health without exceeding the player's MaxHealth.
- Handle multiple targeted players correctly.
- Gracefully handle players who are respawning or whose characters are temporarily unavailable.
- Display a small confirmation notification to the admin after the command succeeds.

--------------------------------------------------
DAMAGE
--------------------------------------------------

Description:
- Damages the specified player(s).

Usage:
damage <target> <amount>

Examples:
damage Kirata 25
damage all 50

Requirements:
- The amount parameter is required.
- Remove exactly the specified amount of health from the targeted player(s).
- Never reduce health below 0.
- Handle multiple targeted players correctly.
- Gracefully handle players who are respawning or whose characters are temporarily unavailable.
- Display a confirmation notification to the admin after the command succeeds.

--------------------------------------------------
KICK
--------------------------------------------------

Description:
- Removes the specified player(s) from the server.

Usage:
kick <target>
kick <target> <reason>

Examples:
kick Kirata
kick Kirata Exploiting is not allowed.
kick all Server restarting.

Requirements:
- Follow the existing admin command framework and coding style.
- Support all existing target selectors.
- If no reason is provided, use the default Roblox-style message:

"You have been kicked by an administrator."

- If a reason is provided, display the following kick message:

"You have been kicked by an administrator.

Reason:
<reason>"

- Everything after the target should be treated as the kick reason.
- Support kicking multiple targeted players.
- Prevent admins from accidentally kicking themselves unless explicitly allowed by the existing permission system.
- Display a confirmation notification to the admin showing which player(s) were kicked.
- Gracefully handle players who leave before the command completes.

--------------------------------------------------
GENERAL REQUIREMENTS
--------------------------------------------------

- Integrate seamlessly into the existing admin system without breaking any existing functionality.
- Follow the existing coding style and project architecture.
- Keep the implementation optimized, reliable, and easy to maintain.

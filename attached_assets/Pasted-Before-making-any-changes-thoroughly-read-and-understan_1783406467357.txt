Create a new chat command:

/t <message>

- Sends a private "Thoughts" message.
- The message should appear as if it were a normal chat message sent by the player, but it is only visible to:
  - The player who sent it.
  - Online admins.
- No other players should ever receive or see the message.

Example:

Player types:
`/t I'm scared...`

The sender and all admins see:

`[THOUGHTS] I'm scared...`

Behavior:
- Preserve the existing chat formatting and styling.
- Display the message in the normal chat feed as if it were a regular chat message.
- Do not display "/t" itself—only the formatted thoughts message.
- The sender should not receive duplicate messages.
- Support all existing markdown formatting (*italic*, **bold**, etc.) if the chat system already supports them.
- Respect the existing language system. If the player currently has a selected language, preserve the existing language behavior while still making the thoughts message visible only to the sender and admins.
- The message should use the same proximity and chat systems as the existing Custom Chat where applicable, except visibility is restricted to the sender and admins.

Chat Logs:
- Every thoughts message must be logged in Chat Logs.
- Display thoughts messages in **purple** so they are immediately distinguishable from normal chat.
- Use the following format:

`[THOUGHTS] [Username]: "I'm scared..."`

Permissions:
- Every player can use `/t`.
- Only the sender and admins can ever see these messages.
- No notification should indicate to other players that a thoughts message was sent.

General Requirements:
- Integrate cleanly into the existing Custom Chat system without rewriting it.
- Follow the existing project architecture and coding style.
- Keep the implementation efficient and avoid duplicate RemoteEvents or listeners.

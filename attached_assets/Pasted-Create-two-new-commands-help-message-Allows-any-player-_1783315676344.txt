Create two new commands:

help <message> - Allows any player (not just admins) to send a help request to online admins.

helpUI - Admin-only command that toggles whether Help Requests are displayed.

Help Command Requirements:
- Every player can use the `help` command.
- The text after `help` is the help request.
- Example:
  - `help I'm bugged`
  - `help I need an admin`
- If no message is provided, notify the player that a message is required.
- Send the help request to every online admin.
- Preserve all existing command functionality.

Help Request Notification:
- Help requests should NOT open in a menu.
- Instead, they should appear as individual notification cards in the bottom-left corner of the screen.
- New notifications should stack upward if multiple requests exist.
- Each notification should animate smoothly into view.
- The notification should have a transparent dark background with rounded corners, matching the style shown in the reference image.

Each notification should display:
- The player's Display Name (large text at the top).
- The player's Username directly below the display name.
- The player's current Team below the username, displayed using the team's color.
- The help message below the team.

Interaction:
- If an admin hovers their mouse over a notification, their camera should temporarily view that player's character.
- The camera should remain focused on that player only while the mouse is hovering over the notification.
- As soon as the mouse leaves the notification, immediately restore the admin's original camera and camera subject.
- Clicking a notification should simply dismiss it for that admin only.
- Dismissing a notification should NOT notify the player who requested help.
- Dismissing it should not affect other admins—they should still see their own copy until they dismiss it.

helpUI Command:
- `helpUI` should toggle whether help notifications are shown for that admin.
- When disabled, new help requests should not appear.
- When enabled again, only future help requests should appear (unless the existing system already stores pending requests).

Additional Requirements:
- Handle multiple simultaneous help requests correctly.
- Remove notifications if the requesting player leaves the game.
- Prevent duplicate errors or memory leaks.
- Follow the existing coding style, RemoteEvents, and admin framework.
- Do not rewrite existing systems—only integrate this feature cleanly into the current project.
